# 万卷灵境 UI 问题清单与治理计划

> 日期：2026-05-30　基于运行时 DOM 实测数据（非主观印象）
> 注：审美/美感判断需人眼确认（当前环境无法读取截图）；本清单聚焦**可量化、可验证**的技术问题。
> 当前界面截图：`ui-audit/current-ui.jpg`（请自行打开查看）

## 一、UI 不一致（"设计感弱"的技术根源）— 高优先级

实测数据证明缺失设计系统：

| 维度 | 实测现状 | 健康基准 | 问题 |
|------|---------|---------|------|
| 按钮样式 | 241 个按钮 / **33 种**不同组合 | 3-5 种 | 每处按钮各写各的 |
| 圆角 | **11 种**（0/4/6/8/10/12/16/999px…） | 2-3 种 | 视觉杂乱 |
| 字号 | **10 种**（8/9/10/11/12/13/14/15/16/18px） | 5-6 级字阶 | 无统一字阶 |
| 颜色 | 110 种硬编码色（主题已覆盖 98.8%） | token 化 | 已基本解决 |

**根因**：无设计 token（圆角/间距/字号/阴影）、无统一按钮组件。
**治理**：① 定义设计 token（CSS 变量）② 收敛圆角→2-3 档、字号→标准字阶 ③ 把 33 种按钮收敛为几种标准按钮样式。通过 ui-overrides.css 注入（零 JS 改动，可控）。

## 二、延迟刷新（"打开界面才刷新出现"）— 高优先级

实测：**47 处 setTimeout + 10 处 requestAnimationFrame**，无懒加载。部分 UI 靠定时器延迟挂载/取数。

可疑位置（节选）：
- `L15694` `setTimeout(()=>requestAnimationFrame(()=>setNodes(...)),120)` — 节点延迟渲染
- `L15713` `setTimeout(()=>fitView(...))` — 画布视图延迟
- `L16145/16407` `setTimeout(()=>saveCanvasState(),80)` — 防抖保存（合理）
- `L16437/16625` `setTimeout(()=>setShouldFitView(true))` — 视图标志延迟

**治理**：逐个甄别——防抖/等动画的保留；纯粹"延迟挂载数据"的改成数据就绪即渲染。需谨慎（有些延迟是故意的）。

## 三、图标质量（"简陋、像直接画的 SVG"）— 中优先级

实测：**30 处手画 inline `<svg>` + 79 个 `<path>`**；lucide 图标库已打包但只用了一部分。

- 多数手画 SVG 其实是 lucide 风格（viewBox 24 / stroke currentColor），不算简陋，只是内联而非组件化。
- 真正需提升的是少数自定义图标（如品牌 logo）+ 风格不统一的图标。

**治理**：① 统一图标尺寸/线宽规范 ② 把内联 SVG 收敛为统一图标组件 ③ 替换确实简陋的自定义图标。

## 四、审美方向（需用户输入）

"高级感"因人而异。需明确：目标风格（极简 / 科技感 / 参考某 app）？这决定设计 token 的取值方向。

## 治理顺序建议
1. **设计 token + 按钮收敛**（收益最直观，CSS 注入可控）
2. **延迟刷新**（技术 bug，逐个验证）
3. **图标统一**（依赖审美方向）

> 所有改动通过 ui-overrides.css 注入或可回滚方式，每步 build + 启动 + DOM 指标对比验证。
